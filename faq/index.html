  • Basics

  • How do I sign up?

    All you need to do is to provide a valid email address and a password. Your account will be ready, however, it will remain in UNVERIFIED status. The registered account must be verified by clicking on the activation link sent via email to the registered email address, without which your access to the data mapper app is blocked.


    Sometimes, the activation email might end up in the spam folder - please check the spam folder for the activation email.

  • Why do I have to sign up?

    Every registered user would be able to maintain his/her own maps (private) catering to a set of source and target files along with the mapping and transformation rules governing the mapping process. These maps would be treated as private resources. The same applies to jobs created by the user, they are treated as private jobs.


    However, one can choose to share a map enabling access to other registered users who can clone a copy to their private workspace and use it as needed

  • Can I create multiple accounts?

    Sure, as long as each account is associated with a distinct email address.


    TIP: If you are using google hosted email or Gmail account, an alias created with a suffix of + sign and a keyword would qualify for a distinct email id

  • What is a map?

    A map is an artifact that you create for transforming a source file in a specific format to a target file format. The process involves defining attribute definition of source and target files at sheet level and a mapping construct that defines rules of attribute mapping between the source and target at sheet level.

  • Can I make changes to an existing map (Edit)?

    Sure, once the map is created it is identified with a unique id. Any changes via the edit map facility will update the associated map on the repository.


    However, be aware that any existing job that refers to the map will now inherit the changes automatically.


    TIP: If you want to maintain versions of the map, you can clone the existing map and create a new map and impart changes on the cloned map

  • What are mapping rules?

    A map contains a collection of rules around attribute mapping and transformation. They can be classified as:


    1) Source Validation Rules

      Applied as a pre-processor validating the source data before applying mapping rules. Any violation would halt the mapping process with appropriate error conditions

        a) Attribute data type definition (String, Date or Number)

        b) Attribute constraints (unique, required)

    2) Target Validation Rules

      Applied as a post-processor validating the mapped result. Any violation would halt the mapping process with appropriate error conditions

        a) Attribute data type definition (String, Date or Number)

        b) Attribute constraints (unique, required)

    3) Mapping Rules


      Mapping rules that identify a source attribute to be mapped to a target attribute. The mapping can be a simple copy mapping or transformations involving computation or processing

    4) Map & Transform Rules


      Governs the transformation process involving computation, aggregation or other forms pre-processing on the source attribute before assigning to a target attribute

  • What is a job?

    An attempt to map a source file to a target file using the rules defined in a map. In other words, execution of a map by providing an input file that conforms to the source file format and attributes definition as defined in the map.


    A created job can be executed any number of times, each time it looks up the current definition of the referred map and produces an output file.

  • Can I make changes to an existing job?

    No. A job once created cannot be modified, but can be executed any time (again).

  • Can I remove a job?

    Yes. A job that is created and owned by you can be deleted at any time. A deletion would remove all associated resources.

  • Design - Mapper (Stage I)

  • What do I need to create a map?

    A map need to be identified with the following:


      a) A unique map name

      b) A source excel file

      c) A target excel file

      d) Basic processing considerations

        i) A row number where the headers are expected to be found on the source excel file

        ii) Should the source and target format is same, a flag to indicate the reuse of source file requirement

        iii) Should the mapping process need to evaluate and consider commonality and differences with the last processed source file, a flag to indicate the diff processing requirement

  • What are the stages in designing a map?

    On the Data Mapper app, designing a map involves four stages:


      1) Map specification - Defining name and map settings

      2) Source Validation Rules - Defining source attribute definition and constraints

      3) Target Validation Rules - Defining target attribute definition and constraints

      4) Mapping Rules - Defining rules for mapping a source attribute to a target attribute

  • Design - Source Validation Rules (Stage II)

  • My source excel has multiple sheets

    Sure, all the sheets of the source excel will be presented for selection/processing on individual tabs. You just need to identify the sheets that contain data to be used in the mapping process. Any sheet that is not considered can be marked as ignored by checking the 'Ignore Sheet' flag on the top of the sheet.


    Sheets marked as ignored will not be presented in source sheets list on the mapping rules definition page. It is just a convenience to keep the interaction focused.

  • What happens if I mark a sheet as 'Ignore'?

    The sheets that are marked as ignored will not be presented for mapping when you build mapping rules.

  • What are preprocessors?

    A preprocessor is a set of rules to be applied on the data from the specific sheet and passed as input for the mapping process. Some rules aid in skipping rows based on conditions and some helps you to massage specific attributes on each row.

  • How many preprocessors can I define on a sheet?

    No restrictions; you can have as many rules as required to accomplish the goal. The rules will be evaluated sequentially in the order they are defined with output of a rule evaluation passed as input to the next rule processing. This can be viewed as a pipeline for rules execution until the last rule is evaluated, resulting in the final dataset that will be exposed to mapping process.

  • What are the preprocessors available?

    Preprocessors on the source validation can be categorized as follows:


      1) Duplicate processing (deduplication)

      2) Skip rows

      3) Date range adjustment

      4) Empty value

      5) Case conversion

      6) Number rounding

  • Can a preprocessor rule be modified?

    Yes, a preprocessor rule can be modified. The rule and or the current rule's parameter settings can be changed any time. Any future reference to this map would reflect the new rules automatically.

  • Can a preprocessor rule be deleted?

    Yes, a preprocessor rule can be deleted any time. Any future reference to this map would reflect the updated ruleset automatically.

  • Why do certain rules not require lookup attribute definition, while most rules do?

    Any rule intended to make changes (update) to the looked up attribute like case conversion, number rounding, conditional skip, deduplication and date range fix would require a valid definition for the lookup. Only exception is 'Skip N Rows' which simply skips the first N rows from processing without any concern over the data.

  • What kind of duplicate processing is supported?

    Based on the specified criteria, the identified duplicate rows can be:


      a) Removed from the result dataset

      b) Obfuscated, meaning suffix the attribute with a running counter so that the resulting dataset of void of duplicates

  • What does skip rules do?

    Skip rules can skip rows based on a criteria or simply the first N rows.


      a) Skip first N rows - Will simply remove the first N rows of the sheet from the final dataset.

      b) Skip a row depending on the empty or not empty condition

      c) Skip a row depending on an equals or not equals condition

      d) Skip a row depending on string starts or ends with condition

  • How does the 'Date range fix' rule work?

    A source sheet may contain one or more date attributes. A date range for the current dataset on the specified attribute can be inferred based on the lower and upper date bounds. A date range fix rule expects a target from and to bounds for a date range.


    With this target date range, algorithmically extrapolate or interpolate a found date value on a given row; thus shifting the dates to a new range.


    This preprocessor is useful when you are trying to shift forward or backwards a set of records based on a date attribute.

  • What does 'Empty Value' rule do?

    This preprocessor simply empties the value of a given attribute on all rows of the source sheet

  • What does a case conversion preprocessor do?

    This preprocessor simply applies the case conversion rule (upper or lower) on the value of a given attribute on all rows of the source sheet when producing result

  • What kind of number processing is supported?

    A source sheet may contain one or more number attributes. A number preprocessor simply helps you update the attribute value to a new number value with the result of a specified rounding (up or down) function.

  • What are the data types supported?

    The mapper app supports String, Date and Number data types. Any other datatype that is not supported should be mapped to a simple String type for convenience. For certain preprocessors, specification of right types is necessary.

  • Why should I specify type for an attribute?

    By default, all the attributes are treated as of string data type. However, if you are planning to apply type specific preprocessors or mapping rules, it is important to specify appropriate data type for an attribute.

  • When should is mark an attribute as 'Required'?

    If you intend to let the mapper validate the source data and check for empty value for an attribute, you should check the 'Required' setting for the attribute.

  • Can I have more than one attribute marked as 'Required'?

    Sure, you can mark any attribute that needs the presence check as 'Required'

  • When should is mark an attribute as 'Unique'?

    If you intend to let the mapper validate the source data and run a unique check on the attribute values, you should check the 'Unique' setting for the attribute.

  • Can I have more than one attribute marked as 'Unique'?

    Sure, you can mark any attribute that needs the uniqueness check as 'Unique'

  • What is the significance of 'Validate' on the Source Validation Rules page?

    The rules configured on the source sheet may require certain dependencies based on the rules applied. For example, a date-oriented rule expects the source attribute to be of date type, number-processing rule would expect the type to be of number. Moreover, the preprocessor rules might also require data type checking. Hence, it is a good idea to run validation before moving to the next stage in the process.

  • What is the significance of 'Save' on the Source Validation Rules page?

    It is an opportunity to save the created metadata so as not to repeat the design process.

  • Design - Mapping Rules (Stage IV)

  • Why I am seeing fewer sheets (tabs) on the mapping rules page?


    Target Sheets marked as ignored will not be presented on the mapping rules definition.

  • What happens if I mark a sheet as 'Ignore'?

    In mapping stage, checking 'Ignore' flag would ignore all the mapping rules defined for that target sheet. The mapping rules defined will not be removed, rather just ignored during execution. This is just a convenience that would help testing of the mapping.

  • What are preprocessors?

    A preprocessor is a set of rules to be applied on the input data to the mapping process. Preprocessors aid in skipping rows based on conditions or massage specific attributes on each row.

  • How many preprocessors can I define on a sheet?

    No restrictions; you can have as many rules as required to accomplish the goal. The rules will be evaluated sequentially in the order they are defined with output of a rule evaluation passed as input to the next rule processing. This can be viewed as a pipeline for rules execution until the last rule is evaluated, resulting in the final dataset that will be exposed as input to the mapping process.

  • What are the preprocessors available?

    Preprocessors on the mapping process can be categorized as follows:


      1) Skip rows

      2) Empty value

      3) Case conversion

  • Can a preprocessor rule be modified?

    Yes, a preprocessor rule can be modified. The rule and or the current rule's parameter settings can be changed any time. Any future reference to this map would reflect the new rules automatically.

  • Can a preprocessor rule be deleted?

    Yes, a preprocessor rule can be deleted any time. Any future reference to this map would reflect the updated ruleset automatically.

  • What does skip rules do?

    Skip rules can skip rows based on a specified criteria.


      a) Skip a row depending on the empty or not empty condition

      b) Skip a row depending on an equals or not equals condition

      c) Skip a row depending on string starts or ends with condition

  • What does 'Empty Value' rule do?

    This preprocessor simply empties the value of a given attribute on all rows of the input dataset

  • What does a case conversion preprocessor do?

    This preprocessor simply applies the case conversion rule (upper or lower) on the value of a given attribute on all rows of the input dataset

  • What is the purpose of specifying a source sheet in the mapping rule configuration?

    Mapping rule configuration facilitates mapping configuration for each of the target sheets (that are not ignored). For each of the target sheet, you can specify the source sheet that serves as input. Note that the data passed would be validated and preprocessed dataset based on the source validation rules defined on the chosen sheet.

  • Would I be able to change the source sheet?

    Yes, you can. However, this would reset the configuration recorded for that sheet and will start afresh. Note that all the attribute mapping defined on this mapping would have access to attributes of the chosen source sheet.

  • Can I choose to focus on a subset of input data for mapping considerations?

    Yes, you can by choosing appropriate scope rule. However, this would reset the configuration recorded for that sheet and will start afresh.

  • What are scope rules?

    Scope rules helps identify a subset of the input data for mapping process.

      1) All Rows

        Consider processing all rows of the input dataset.

      2) Distinct Rows

        Filter the input dataset for rows with distinct value for specified attribute. If multiple rows are found for a specified value, the first row would be considered for processing.

      3) Distinct Composite Rows

        Filter the input dataset for rows based on distinct value derived from the composite columns (a value composed by a simple concatenation of attributes). If multiple rows are found for a specified value, the first row would be considered for processing. This is useful when trying to aggregate or derive statistical values from the dataset.

  • Would I be able to change the scope rule?

    Yes, you can. Be aware of the impact this scope change will have on mapping rules defined at attribute level within the map as they may be impacted; especially any mapping rule involving aggregation and custom rules.

  • Can I reset and start fresh on the scope rule configuration?

    Yes. Simply click on the trash icon next to the scope rule setting to reset to start fresh.

  • Is it necessary to map all attributes in a mapping?

    No. You can choose to ignore mapping for attributes that are not expected to be part of the output. In other words, any attribute that is not identified with a mapping rule will be ignored.

  • What are the mapping options available for an attribute?

    Mapping rule identifies how the output value will be computed. Following types of mapping rules are supported


      1) Simple

        A simple value-for-value mapping where a designated source attribute is mapped to a target attribute.

      2) Conditional

        The output value is derived based on a conditional evaluation.

      3) Switch

        The output value is derived based on the matching condition on a switch construct.

      4) Computed

        The output value is computed based on a outcome of executing a set of operations on source attributes.

      5) Aggregated

        The output value is derived based on the defined aggregation operation.

      6) Custom

        A pure custom logic represented as a block of code that will be executed at runtime.

  • Tell me about attribute mapping type 'Simple Mapping'?

    A Simple Mapping rule facilitates assignment of either a static value or a looked up value from the source sheet to a target attribute.


    Few pointers on Simple Mapping:


      Static value is a simple string literal

      A Lookup value is a value picked from the dataset for the specified source sheet and attribute.

        Only attributes from the source sheet specified in the mapping configuration will be exposed.

        Keep in mind the data types of the source and target attributes should match, otherwise will result in error.

      You can switch from static to lookup mode as desired any time.

      You can edit (change) the mapping mode as desired any time.

      Only one assignment rule can be specified for Simple Mapping.

      If Finally post-processor is specified, the mapped value would be processed with the custom logic before assigning to the target attribute.

  • Tell me about attribute mapping type 'Conditional Mapping'?

    A Conditional Mapping rule facilitates conditional assignment to a target attribute.


    A Conditional Mapping is made up of:


      One or more logical conditions defined on the source attributes (any valid attribute on the source sheet).

      A TRUE and FALSE setting invoked based on the outcome of the condition(s) evaluation and assigns a value to the target attribute.

        Static Value - A static literal

        Lookup Value - A looked up value from the source sheet

        Empty Value - A empty value (forcefully set to empty value)

        Skip Row - Skip the row from the mapping process (use with caution)


    Few pointers on Conditional Mapping:


      Static value is a simple string literal

      A Lookup value is a value picked from the dataset for the specified source sheet and attribute.

        Only attributes from the source sheet specified in the mapping configuration will be exposed.

        Keep in mind the data types of the source and target attributes should match, otherwise will result in error.

      You can switch between assignment modes for the TRUE and FALSE conditions as desired any time.

      You can edit (change) the conditions set as desired any time.

      You can have multiple conditions specified and they will be evaluated as logical unit with a logical operator that combines the result (AND or OR) to arrive at the final evaluated result

  • What are the condition types supported in 'Conditional Mapping'?

    A Conditional Mapping could employ one or more conditions each evaluating a logical condition. Following is the list of operations supported to build a condition:


      a) Empty Condition - Check if the input value is empty or not

      b) Equals Condition - Check if the input value is equals or not equals to a specified value (static or looked up value)

      c) Arithmetic Condition - Compare the input value with the specified value (static or looked up value) using arithmetic operations like <, <=, > or >=

      d) String Operations - Check if the input value contains the specified value (static or looked up value), equivalent to substring operation

  • What are the available modes of combining conditions in 'Conditional Mapping'?

    Conditions are sequentially executed and logically evaluated based on the logical operator - AND and OR


    NOTE: The last condition in the sequence must not be configured with any logical operator

  • Tell me about attribute mapping type 'Switch Mapping'?

    A Switch Mapping rule facilitates conditional assignment to a target attribute based on the input vale.


    A Switch Mapping is made up of:


      One or more conditions that will be sequentially evaluated

      Upon hitting a condition that evaluates to TRUE, the output attribute will be and assigns a value to the target attribute.

        Static Value - A static literal

        Lookup Value - A looked up value from the source sheet

        Empty Value - A empty value (forcefully set to empty value)

        Skip Row - Skip the row from the mapping process (use with caution)


    Few pointers on Switch Mapping:


      Static value is a simple string literal

      A Lookup value is a value picked from the dataset for the specified source sheet and attribute.

        Only attributes from the source sheet specified in the mapping configuration will be exposed.

        Keep in mind the data types of the source and target attributes should match, otherwise will result in error.

      You can switch between assignment modes for the switch conditions as desired any time.

      You can edit (change) the conditions set as desired any time.

      The first condition in the sequence of conditions specified that evaluates to TRUE, will be executed for assigning a value to target attribute

  • What are the condition types supported in 'Switch Mapping'?

    A Switch Mapping could employ one or more conditions each evaluating a logical condition. Following is the list of operations supported to build a condition:


      a) Empty Condition - Check if the input value is empty or not

      b) Equals Condition - Check if the input value is equals or not equals to a specified value (static or looked up value)

      c) Arithmetic Condition - Compare the input value with the specified value (static or looked up value) using arithmetic operations like <, <=, > or >=

      d) String Operations - Check if the input value contains the specified value (static or looked up value), equivalent to substring operation

  • Tell me about attribute mapping type 'Computed Mapping'?

    A Computed Mapping rule facilitates assignment of a computed value from one or more source attributes to a target attribute


    A Computed Mapping is made up of:


      One or more conditions defined on the source attributes (any valid attribute on the source sheet).

      Conditions will be evaluated sequentially and output of a condition will be passed as input to the next condition

      Final resulting value from the last condition will be and assigns a value to the target attribute.


      A Condition can have input value as:

        Static Value - A static literal

        Lookup Value - A looked up value from the source sheet


    Few pointers on Computed Mapping:


      Static input value for a condition is a simple string literal

      A Lookup input value for a condition is a value picked from the dataset for the specified source sheet and attribute.

        Only attributes from the source sheet specified in the mapping configuration will be exposed.

        Keep in mind the data types of the source and target attributes should match, otherwise will result in error.

      You can switch between assignment modes for the switch conditions as desired any time.

      You can edit (change) the conditions set as desired any time.

      You can have multiple conditions specified and they will be evaluated in sequence with an operator that combines the results to arrive at the final evaluated result

  • What are the operator types supported in 'Computed Mapping'?

    A Computed Mapping could employ one or more conditions each executing an operation. Following is the list of operations supported to build a condition:


      a) String Operations - Case conversion, Concatenation and Substring

      b) Arithmetic Operations - Add, Subtract, Multiply and Divide

  • What are the operator types supported in Computed Mapping while combining multiple conditions?

    A Computed Mapping could employ one or more conditions each executing an operation. The final result is arrived by combining the results of each operation in sequence. Following is the list of operations supported while combining conditions condition:


      a) String Operations - Concatenation

      b) Arithmetic Operations - Add, Subtract, Multiply and Divide

  • Tell me about attribute mapping type 'Aggregated Mapping'?

    An Aggregated Mapping rule facilitates assignment of a computed statistical value from one or more source attributes to a target attribute


    An Aggregated Mapping is made up of:


      One or more conditions defined on the source attributes (any valid attribute on the source sheet).

      Aggregate conditions will be evaluated sequentially and output of a condition will be passed as input to the next condition

      Final resulting value from the last condition will be and assigns a value to the target attribute.


        A Condition can have input value as a looked up value from the source sheet


    Few pointers on Aggregated Mapping:


      A Lookup input value for a condition is a value picked from the dataset for the specified source sheet and attribute.

        Only attributes from the source sheet specified in the mapping configuration will be exposed.

        Keep in mind the data types of the source and target attributes should match, otherwise will result in error.

      You can edit (change) the conditions set as desired any time.

      You can have multiple conditions specified and they will be evaluated in sequence with an operator that combines the results to arrive at the final evaluated result

  • What are the operator types supported in 'Aggregated Mapping'?

    An Aggregated Mapping could employ one or more conditions each executing an operation. Following is the list of operations supported to build a condition:


      a) Aggregate Operations - Sum, Average, Min and Max

      b) Statistical Operations - Mean, Median, Variance, Standard Deviation and Percentile

  • What are the operator types supported in Computed Mapping while combining multiple conditions?

    A Computed Mapping could employ one or more conditions each executing an operation. The final result is arrived by combining the results of each operation in sequence. Following is the list of operations supported while combining conditions condition:


      Arithmetic Operations - Add, Subtract, Multiply and Divide

  • Tell me about attribute mapping type 'Custom Mapping'?

    A Custom Mapping rule facilitates execution of a block of code and assign the result to the target attribute


    NOTE: Please refer to the restrictions and conventions to be followed while writing custom logic as code for use in 'Finally' and 'Custom' mapping option.

  • What is the purpose of 'Finally' in the attribute mapping?

    Finally option allows you to specify a custom code to be run as a post-processor on the mapped output before assigning to the target attribute. This is optional, if not present the mapped output will be assigned directly to the target attribute.


    Conventions to be followed:

      a) Input Data

        Source data is accessible via a simple array notation like:

          dataSource['SHEET_NAME']['ROW_NUMBER']['ATTRIBUTE_COLUMN_KEY_IN_EXCEL'] -

            Will get you access to input dataset as returned by the source validation phase.

            A cell value on the source dataset can be fetched by passing appropriate values for SHEET_NAME, ROW_NUMBER and ATTRIBUTE_COLUMN_KEY_IN_EXCEL

          _data['SHEET_NAME']['ROW_NUMBER']['ATTRIBUTE_COLUMN_KEY_IN_EXCEL'] -

            Will get you access to raw source data (before source validation).

            A cell value on the source dataset can be fetched by passing appropriate values for SHEET_NAME, ROW_NUMBER and ATTRIBUTE_COLUMN_KEY_IN_EXCEL

      b) Custom code

        You are free to write custom logic in Javascript in the block and return a computed value

      c) Return value

        The return value of the block of code is expected to be primitive value - not a collection or object. The returned value will be assigned to the target attribute.

  • Repository

  • What are private and shared maps?

    Maps created by you by default are private. You can choose to share them with others for reference purposes.

  • Can I make changes to an existing map?

    Yes, you can reopen (edit) an existing map and make changes.

  • How do share a map with others?

    Either you can mark the map as shared or export and share the map archive offline.

  • How do I create a copy of a map?

    Simply clone a map from your own or shared repoisory or import a map archive (downloaded via export)

  • What can I do with a shared map?

    You would be able to clone a copy of the map into your repository or download as a archive file for sharing. You cannot edit a shared map, only the owner can make changes.

  • I no longer need a map, can I delete it?

    Yes. Please note that a delete operation would delete the map as well as any existing jobs that are utilizing the map. Use with caution!

  • What does an downloaded map archive contain?

    A download action exports the map configuration as an archive file that can be saved to your local file system. The archive file contains:


        1) A map configuration file in JSON format containing meta data that represents the map configuration and rules

        2) An excel file that was used as the source template in the map design process

        3) An excel file that was used as the target template in the map design process

  • How do I import a map archive file?

    Use the Import function in the repository under private maps

  • Jobs

  • What is a job?

    A Job is an execution unit that applied a mapper to produce mapped output after applying specified mapping rules and transformations

  • How do I run a job?

    A Job can be triggered for execution by clicking on the play icon on the job row. A job can be run any numbe of times, each run will pick up the current map configuration that exists on the repository.

  • How do I find whether a job ran successfully or not?

    Each job row (record) will indicate a status representing:


        a) RUNNING - Indicating the job is currently running

        b) SUCCESS - Job completed successfully

        c) FAILED - Job execution failed

  • How do I find the job execution details?

    Every job execution captures the results in a log file. The log file can be viewed for details.

  • What does an downloaded job archive contain?

    A download action exports the job configuration as an archive file that can be saved to your local file system. The archive file contains:


        1) A job configuration file in JSON format containing meta data that represents the job configuration

        2) An excel file that was used as the input for the job execution

        3) An excel file that was produced as a result of execution

        4) A log file with informational and error log entries captured during execution

  • I no longer need a job, can I delete it?

    Yes. All the artifacts related to the job will be deleted.
